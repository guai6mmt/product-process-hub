# 产品流程与文件管理系统 · 本机版 v2

为"产品全流程的文件管理"做的小系统：按「产品类型 × 流程」配置标准清单，自动展开成每个产品的文件清单，并跟踪上传 / 消保 / 法审 / 用印的进度与留痕。
逻辑参考上级目录的 [`产品流程与文件管理_逻辑规格.md`](../产品流程与文件管理_逻辑规格.md)。

零依赖（仅 Python 标准库）、单文件后端 + 单文件前端、数据全在本机。

---

## 怎么运行

- **双击 `run.bat`** 即可（自动找 Python 并打开浏览器 http://localhost:8765）。
- 或命令行：`python app.py`（加 `--no-browser` 则不自动开浏览器）。
- 前提：装了 Python 3（`python --version` 能看到版本即可），**不需要任何第三方库**。
- 关闭：在黑色命令行窗口按 `Ctrl + C`，或直接关掉窗口。
- 端口 `8765` 被占用会启动失败——先关掉占用它的程序（比如另一个本系统实例）再启动。

## 核心概念（一次配置，处处派生）

| 概念 | 说明 |
|---|---|
| **流程** | 产品的生命周期阶段：产品准入 → 销售准入 → 上架发售 → 存续管理（可在"模板管理"里增删）。 |
| **产品类型 × 流程 模板** | 标准文件清单按「产品类型 × 流程」配置；某类型在某流程没配时，**自动继承"通用"**（产品类型也可增删）。 |
| **一键开产品（实例化）** | 填几个参数，系统据模板自动展开成这个产品的整张文件清单。 |
| **派生状态** | 你只手动维护 4 类原子事实（是否上传 / 消保 / 法审 / 用印），"是否到位""下一步动作""进度"全部自动算。 |
| **审查留痕** | 每次审查状态变更都写入 `review_log`（含时间与意见），只增不删。 |

## 它能做什么（飞书做不到的那几件）

1. **模板管理 = 总览矩阵 + 聚焦编辑器**
   - 行＝产品类型、列＝流程的矩阵，一眼看清哪格自定义（`N 项`）、哪格`继承通用`、哪格`未配置`。
   - 点格子进入聚焦编辑：文件项按大类分组，审查要求用标签展示（消保/法审/用印·方）、纳入条件只在非"总是"时出现（有海报时/仅合并版/仅拆分版）。
   - **继承可微调**：继承格只读展示"通用"清单，点「复制【通用】清单到此」即 fork 成本类型专属，随后可**自由增 / 删 / 改**文件项；「恢复为继承通用」一键还原（不影响已建产品）。
   - 矩阵里流程与产品类型均可 **× 删除**（"通用"为基线不可删；删流程会连带清理它在所有产品中的文件项）。
   - 每个文件项可设 **"是否需要上传主文件"**；需要但未上传时，详情页状态条会显示"主文件 未上传"提示。

2. **一键开产品**：填产品名 + 产品类型 + 补充协议份数 + 有无海报 + 合同形态（合并/拆分），自动生成各流程的整张清单（消保/法审/用印要求已预填）。

3. **工作流详情页**
   - **折叠行状态条**：每个文件项一眼看清走到哪步——`✓ 已完成`、`● 当前待办`（蓝）、`● 送审中`（琥珀）、`● 退回`（红）、`○ 未解锁`（灰）。
   - **一键处理（CTA）**：点右侧"下一步动作"徽章，直接拉起上传 / 展开并定位到对应审查块。
   - **只看未到位** + **展开/收起全部**：清单一多也能快速聚焦。

4. **流程顺序为软提示（不强制）**：前序流程未备齐只在顶部给提示，不拦截后续操作；消保 / 法审 / 用印各状态可独立设置，互不卡。

5. **退回必须填"退回意见"**：状态切到"退回"时强制填写原因，记入留痕，并显示在该审查处（合规刚需）。

6. **退回派生新版本**：上传新版本保留旧版本，并自动重置审查链重走。

7. **文件入库 + 按结构导出**：上传的文件存在本机 `data/files/`，可随时下载；可按"产品/流程/大类/文件项"目录结构导出 zip（含 `_清单.csv`）。

8. **驾驶舱 + 今日待办**：每个产品的备齐进度、每个文件的"下一步动作"自动汇总。

9. **跨产品复用文件**：详情页每个流程有「📥 从其他产品导入」——把另一个产品**相同流程、同名文件项**的已上传文件与审查状态**一次性复制**过来（之后各自独立）。适合同一发行人的多只产品文件一致的情形。

10. **共用文件库**：顶部「文件库」可维护**单个文件**和**文件包（一套：主文件＋消保＋法审＋用印）**。在文件项里：用「📚库」选用单个文件；用「📦 从文件库导入整套」一键导入整包，并自动把消保 / 法审 / 用印置为"已通过 / 已用印"。

## 规则要点（与规格一致）

- 代销协议（总对总 + 补充协议）：消保 + 法审 + 用印（双方）。
- 委托代销函：仅对方机构用印。
- 宣传材料（PPT/一页通/海报/双录话术/线上智能播报）：消保。
- 合同：合并 1 份 或 拆分 3 份（开产品时二选一）；消保 + 法审。
- 海报：开产品时勾选"含海报"才纳入，纳入时才计入"是否备齐"。
- 标准模板可在"模板管理"里随时改，改的是规则，新开的产品自动跟着变。

## 数据与备份

- 全部数据在本目录的 **`data/`** 文件夹：
  - `app.db`：结构、模板、产品、清单、状态、审查留痕。
  - `files/`：上传的文件本体。
- **备份 = 复制整个 `data/` 文件夹**。换电脑把它拷过去即可（该目录不进版本库）。

## 技术说明

- 后端 `app.py`：Python 标准库 `http.server` + `sqlite3`，无第三方依赖。
- 前端 `index.html`：单文件，原生 JS，无构建步骤。
- 数据表：`process / product_type / blueprint_item`（模板）、`product / requirement / docfile`（实例与文件）、`review_log`（审查留痕）、`lib_package / lib_file`（共用文件库）。
- 数据库结构变更自带迁移（启动时自动给旧库补 `needs_file` 列等），旧数据可平滑升级。
- 自带 v1→v2 数据迁移；模板/流程/类型缺失会在启动时按种子补齐。
